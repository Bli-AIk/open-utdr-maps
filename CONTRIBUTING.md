# 贡献指南

感谢你对 open-utdr-maps 的关注！以下是参与贡献的说明。

## 贡献类型

### 1. 数据整理
清理调试对象、修复瓦片对齐、重组图层命名。

### 2. 逻辑图层
为地图添加碰撞边界、触发器、NPC 出生点等对象图层。

### 3. 世界拼接
创建 Tiled `.world` 文件，将相邻房间连接为完整区域。

### 4. 文档完善
改进地图描述、区域指南、使用教程等。

### 5. 工具改进
增强转换器、浏览器或分析工具。

## PR 要求

提交 PR 时请包含：

- **变更描述** — 做了什么，为什么
- **地图来源** — 标注修改的房间和数据集
- **使用的工具** — Tiled 版本、脚本等
- **截图** — 视觉变更时必须提供前后对比
- 合并前需至少一位审核者通过

## 图层命名规范

请参阅 [docs/layer_spec.md](docs/layer_spec.md)。

整理版地图应尽量遵循标准图层命名，但 `raw/` 目录下的原始地图**不应修改**。

## 注意事项

- `raw/` 目录由自动化工具生成，**请勿手动修改**
- 整理版地图放在 `curated/` 对应目录下
- 不要提交游戏的二进制文件（`data.win` 等）
- 不要提交 GameMaker 反编译代码
