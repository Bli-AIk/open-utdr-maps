# Layer Naming Conventions

All curated maps should follow these layer naming standards. Original maps in `raw/` retain auto-generated layer names.

## Tile Layers

| Layer Name | Description |
|------------|-------------|
| `ground` | Base terrain tiles |
| `decoration` | Decorative overlay tiles |

If multiple layers of the same type exist, use numeric suffixes: `ground_1`, `ground_2`, etc.

## Object Layers

| Layer Name | Description |
|------------|-------------|
| `collision` | Collision boundaries |
| `interaction` | Interactive areas |
| `npc_spawn` | NPC spawn points |
| `trigger` | Event triggers |
| `event` | Story/cutscene markers |

## Auto-generated Layers

These layers are auto-generated by `gm2tiled` and may be kept or renamed during curation:

| Layer Name | Description |
|------------|-------------|
| `views` | Camera view areas |
| `instances` | GameMaker object instances |
| `Tiles_<depth>` | GMS1 tile layers named by depth value |
| Original name | GMS2 native layers keep their original name |

## Layer Order

Top to bottom (foreground to background):

1. Object layers (collision, trigger, etc.)
2. Foreground decoration (`decoration`)
3. Base terrain (`ground`)
4. Views (`views`)
